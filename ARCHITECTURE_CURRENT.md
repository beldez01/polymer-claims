# Polymer Claims — Current Architecture (the short, true version)

> One-page map of what is **active**, what is **frozen**, and what is **user-gated/future**. Runtime state as of 2026-06-26 (last feature merge: local transparency-log layer; networked Rekor backend designed but tabled); doc refreshed 2026-06-26.
> For the detailed phase-by-phase build log, see `docs/superpowers/CONTINUE.md` (continuity log — dense by design).
> For terminology, see `GLOSSARY.md`.

## The system in one line

A **compiler + runtime for science**: the grammar defines *what a claim is*; the protocol defines *how a corpus of claims evolves toward truth*; the local node *hosts* a running corpus; the viewer *renders* the evolving topology.

```
grammar  →  protocol            →  node (src/polymer_claims)  →  viewer
"what a     "how a corpus           a local mutable host           renders the live
 claim is"   evolves" (run_cycle      (NodeRunner + serve)          topology over SSE
             flywheel + daemons)
```

**Purity invariant:** `grammar/` and `protocol/` are pure/deterministic (no clock/random/IO; time-like inputs are passed in). The ONLY impure piece is the umbrella node/server (`NodeRunner` owns the loop/clock; the server owns the network). The `Corpus` is exactly 4 collections.

**Standards skin — attestation export (North-Star arc 2, slice 1, 2026-06-21):** `src/polymer_claims/attestation.py` + the `export-attestation <corpus>` CLI re-express a LICENSED run as the trust standards that already exist — a deterministic **in-toto Statement v1 / SLSA Provenance v1** attestation per LICENSED claim **+ a GA4GH DRS** object per dataset, keyed by the content-address the gate already computes (`dimnames_hash`/`profile_hash`/`semantic_run_id`); the air-gap credential pair are the SLSA `builderDependencies`. The serializer is pure (typed frozen `_Model` DTOs, stdlib `json`/`hashlib`, no clock/signing/network — `contract_index` + adapter registry injected); `resolve_contract_index` is the only IO. Additive/byte-identical when off. **Slice 2 (arc 2, 2026-06-21):** `--format dsse` added — emits per-claim unsigned DSSE NDJSON (`DsseEnvelope`, `signatures: []`); reuses Statements via `build_attestation_statements`; records refactored into `AttestationRecord` + `build_attestation_records` (bundle output byte-identical, golden-captured). Signing/Rekor/DSSE PAE = slice 3. Later slices: WES/TRS/RO-Crate. **Slice 3 (2026-06-25): local ed25519 DSSE signing SHIPPED** (`feat/dsse-signing`) — DSSE PAE + `keygen`/`verify-dsse` + opt-in `--key` on `certify`/`export-attestation` (`[sign]` extra); the Sigstore/cosign/**Rekor** transparency-log layer remains open. **Kernel-proof reproducibility (2026-06-25): `verify-kernel` (H0.1, synthetic offline, CI-guarded) + `verify-kernel --real` (H0.1b, real `@2` content-address-parity gate — merged `32670bb`)** make the n-DMP proof re-runnable; the real pins were captured + verified 2026-06-25 (`scripts/bootstrap_real_kernel_pins.py` two-mode diff clean) and `verify-kernel --real` returns `LICENSED @ REPRODUCED` (378,894 probes; 115,405 DMPs; e-value=∞; IDH-mut n=36) — H0.1b fully closed. Detail in `CONTINUE.md` / `docs/superpowers/2026-06-23-remaining-roadmap.md`.

**Local-node hardening (2026-06-05):** the live server is bounded and serialized for hours-long local runs — `--max-frames` ring retention, an `asyncio.Lock` serializing ticks, bounded SSE queues, and a **non-loopback bind guard** (`serve --host` other than loopback refuses unless `--unsafe-remote-control` is passed). It is still **local-only**: the mutating routes (`/step`/`/pause`/`/resume`) are unauthenticated by design. Real auth/multi-tenant/deploy is the future federated step, not shipped.

**Agent-driven live node (2026-06-06):** the real `LLMGenerationAdapter` (the `[llm]` extra, Anthropic-backed) is built behind the existing generation-bus seam and can now drive the **live node** itself via `serve --llm` (flags: `--llm-model`, `--llm-every N`). The `every_n_ticks` throttle wrapper (`src/polymer_claims/throttle.py`) calls the inner proposer on the 1st tick then every Nth tick, making a live LLM loop watchable and affordable. The LLM adapter runs alongside the seed corpus's existing rival/revision proposers; the universe stays lively on quiet ticks. (The `builtin::const` LLM-asserted-value substrate this adapter started on has since been superseded by real execution adapters and the methylation apparatus — see *Real computation & the epistemic core* below.)

## Real computation & the epistemic core (Phase 2 — built since 2026-06-08)

The system no longer licenses only on asserted values. Three arcs landed:

- **Real execution adapters (2026-06-08)** — claims compute a two-group mean difference from a bundled dataset via two genuinely independent stdlib adapters and license/reject on the **computed** value; `serve --real-data` runs the live node on them. The adapter trust registry (below) bites: a same-owner pair is held PENDING.
- **CES-0 → CES-4 (2026-06-10 → 12)** — the credibility-evidence spine: a content-addressed `AnalysisProfile` apparatus (CES-0) → a DRS-shaped SE-Contract data seam (CES-1) → the first claim to **license on a value computed from a methylation matrix** (CES-2, region Δβ, two methodologically-independent legs) → a license that records its **full content-address** — dataset `dimnames_hash` + apparatus `profile_hash` + `semantic_run_id` (CES-3) → all of it **running live** in `NodeRunner`/`serve` with a drift daemon that re-opens content-drifted licenses (CES-4). **EARNED (2026-06-17, Phase A):** the genome-wide **n-DMP count** now licenses at **REPRODUCED** on a **real TCGA-LAML HM450 cohort** (IDH-mut vs WT; 50,339 DMPs vs an 18,945 null floor; e-value → ∞; two independent legs agree; full content-address recorded). The recomputable-public tier is *earned, not just exercised*, for this reduction. **Region-Δβ attempted on real data (2026-06-17) via held-out top-10k selection (`split_select.py`) — gate WITHHELD at n=10** (held-out e-value 0.867 < pre-registered τ=0.10 → PENDING): the discovery/test split correctly refused a license the naive in-sample test would have granted; power-limited (n=10 → ~5/split), not refuted. REPLICATED stays synthetic (needs a 2nd real cohort). Run caveats: IDH-mut n=10 (conservative — uncovered cases default WT, diluting the contrast); sex-chrom QC skipped (Xena betas carry no chr/pos); data local-only, gitignored.
- **Phase 2.1 → 2.4 (2026-06-12 → 13) — the e-value epistemic core.** Licensing, the corpus FDR budget, and defeat are now **one mechanism**: the FDR ledger is an **e-LOND** online process (FDR control under arbitrary dependence), the native evidence atom is a **Waudby-Smith-Ramdas betting e-value**, and `LICENSED ⇔ adapter-agreement ∧ SATISFIED ∧ grounded ∧ live e-LOND discovery`. A successful **defeat de-licenses through the ledger and refunds** the discovery (`FDRTest.retracted` tombstone). The 4-way gate runs **live** in the node (2.3), and the drift path preserves `LICENSED ⇒ a live discovery` (2.4). grammar/protocol stay pure + numpy-free; Corpus stays 4.
- **§2E tiered independence (2026-06-14) — REPRODUCED / REPLICATED.** An additive `Licensing.independence_tier` (default REPRODUCED → byte-identical). A claim reproduced across **≥2 cohorts with distinct datasets** (distinct `dimnames_hash`) licenses at **REPLICATED**, and its e-LOND test uses the **product** `e₁·e₂` — *one* test/discovery, no α-budget double-count. Cohort B enters umbrella-side via a threaded `run_cycle(replications=)` map (umbrella `replication.py` air-gaps a 2nd **synthetic** cohort and emits the extra satisfaction + product e-value). Deferred follow-ups: a viewer REPLICATED badge and live-node (`NodeRunner`) wiring.
- **Reinstatement → PENDING + n-DMPs-at-FDR (2026-06-14).** Two more epistemic slices. *Reinstatement* is the symmetric counterpart to Phase-2.2 defeat: a `RejectionReason` marker + an INTEGRATE reinstatement pass reopen a *defeat-rejected* claim to **PENDING** (to re-test) when its attacker falls; *refuted* claims stay terminal. *n-DMPs-at-FDR* is a second methylation reduction — the count of differentially-methylated probes licenses on a one-sample count-enrichment betting e-value, with two pooled-t legs agreeing on the integer count (air-gap); umbrella-only, grammar/protocol untouched.
- **Live spectral layout + Procrustes alignment (2026-06-15).** The signed-Laplacian eigenmap (`embedding.py`) is now the **live `NodeRunner` default layout**, orthogonal-Procrustes-aligned each frame to the previous so the universe grows smoothly instead of thrashing on eigenbasis sign/rotation ambiguity. `serve --layout {spectral,force}`; `force` is byte-identical to the prior path; the embedder is lazy-imported so `node.py`'s base import stays numpy-free. Umbrella + viewer only; protocol untouched. (Spec `docs/superpowers/archive/specs/2026-06-15-procrustes-embedding-alignment-design.md`, plan `docs/superpowers/archive/plans/2026-06-15-procrustes-embedding-alignment.md`; usage/theory notes `docs/spectral-layout-*.md`.)
- **Real-data hardening + Phase D (2026-06-17 → 19).** (a) **n-DMP EARNED** at REPRODUCED on a real TCGA-LAML HM450 cohort (Phase A). (b) **IDH-source swap → `tcga_laml_idh@2`** — IDH calls upgraded from dilution-prone GDC open MAFs (n=10) to cBioPortal complete genotyping (**n=36**); region-Δβ re-run at proper power moved the held-out betting e-value **0.867 → 5.672**, still **PENDING** below the e-LOND first-test bar 1/α₁=32.9 (τ fixed). ## v1.2 (moved out of the repo)

The frozen v1.2 FormalClaim ecosystem (the `polymer_formalclaim` IR package, the 47-claim corpus, the `claim-harness` Claude Code plugin, schema, legacy workflows) was **moved out of this repository on 2026-06-17** — preserved locally, pending eventual deletion. Nothing in the active runtime is affected by its removal.
